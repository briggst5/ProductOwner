# Solution Demo Prep

## Steps

1. List **PI Objectives** or sprint goal
2. Query **completed Features/Stories** since last demo (Closed, demo-able)
3. Map demo flow: objective → feature → story demo sequence
4. For each demo segment:
   - **Presenter** (team/role)
   - **Setup** required
   - **Script:** "We committed to… [objective]. Here we show… [capability]."
   - **Acceptance criteria** hit — bullet proof points
5. Note **gaps** — committed but not demo-able (risk for I&A)
6. Stakeholder Q&A parking lot

## Output

```
## System Demo Runbook — [ART/Team] — [date]

**Duration:** [min]
**Audience:** [stakeholders]

### Flow
| # | PI Objective | Demo | Presenter | Time |
|---|--------------|------|-----------|------|

### Demo script
#### Segment 1: [Title]
- Setup: ...
- Narrative: ...
- AC shown: ...

### Not ready to demo
| Item | Reason | Mitigation |

### Q&A anticipated
- ...
```

Align with `release-notes-draft` for post-demo communications.
