# Inspect & Adapt Synthesis

Ceremony: `reference/wiki/agile-practices/safe-agile/agile-ceremonies-safe.md`

## Inputs

1. **Quantitative:** `iteration-health-check` across all PI iterations — predictability, spillover, flow
2. **PI Objectives:** committed vs achieved (business value delivered)
3. **Qualitative:** retro notes, SM/Coach Sync themes (user-provided)
4. **System Demo feedback** (user-provided)

## Steps

1. Compile metrics table for ART
2. Identify **top 3 wins** and **top 3 problem areas**
3. Run problem-solving structure:
   - Problem statement
   - Fishbone categories (People, Process, Tools, Dependencies)
   - Root cause (5 Whys — draft if user supplies facts)
4. Propose **improvement backlog items** — specific, owned, fit in next PI
5. Prioritize improvements (WSJF or simple impact/effort)

## Output

```

### Quantitative summary
| Metric | PI value | Trend | Target |

### PI Objectives scorecard
| Objective | BV planned | BV delivered | Met? |

### Problem-solving

**Root cause:** ...
**Improvement items:**
| # | Action | Owner | Target iteration |

### Carry forward to next PI
- ...
```

Feed improvement items to ADO as Features/Stories only with user approval.
